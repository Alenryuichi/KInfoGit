EXECUTIVE SUMMARY
════════════════════════════════════════════════════════════════════════════════

The AI Daily feature is a static, zero-backend curated news digest system that:

1. ✅ Takes Horizon's markdown summaries as INPUT
2. ✅ Classifies items into 3 sections using keyword matching
3. ✅ Stores tags, scores, sources in structured JSON format
4. ✅ Renders via React + Next.js static generation
5. ✅ Deploys to GitHub Pages as pure HTML

No API routes, no database, no SSR - just static HTML files generated at build time.

SECTION_KEYWORDS defines 3 sections with associated keywords:

  ]

    'novel', 'benchmark', 'scaling', 'arxiv', 'breakthrough'
  ]

  engineering: [
    'tool', 'library', 'framework', 'tutorial', 'guide', 'open-source',
    'CLI', 'SDK', 'API', 'web tool', 'unicode', 'macOS', 'CPU', 'debug'
  ]

Algorithm (Lines 57-72):

  function classifyItem(item) {
    // Step 1: Combine all text fields
    const text = `${item.title} ${item.summary} ${item.tags.join(' ')}`.toLowerCase()

    // Step 2: Count keyword matches
    let bestSection = 'headlines'
    let bestScore = 0
    
    for (const [section, keywords] of Object.entries(SECTION_KEYWORDS)) {
      const score = keywords.filter(kw => text.includes(kw.toLowerCase())).length
      if (score > bestScore) {
        bestScore = score
        bestSection = section
      }
    }
    
    // Step 3: Return winning section
    return bestSection
  }

Classification Examples (from 2026-04-10):

  "FBI used iPhone notification data to retrieve deleted Signal messages"
  → Found: 'FBI' (headlines keyword)

  → Classification: RESEARCH ✓

  "GitButler raises $17M to build next-generation version control"
  → Keywords: 'raises' (headlines), 'tool' (engineering)
  → Classification: ENGINEERING ✓ (highest match count)

  **Background**: ...
  
  <details><summary>References</summary>
    <ul>
      <li><a href="...">Reference</a></li>
    </ul>
  </details>
  
  **Discussion**: ...
  
  **Tags**: `#tag1`, `#tag2`
  
  ---

Parsing Details (convert-horizon.ts Lines 76-184):

  State Machine Uses:
    - currentItem: Current NewsItem being built
    - currentField: 'summary', 'background', 'discussion'
    - fieldContent: Array of lines for multi-line fields

  Regex Patterns:
    
    Item header:
      /^## \[(.+?)\]\((.+?)\)\s*⭐️\s*([\d.]+)\/10/
      Captures: [1]=title, [2]=url, [3]=score
    
    Source line:
      /^(\w+)\s*·\s*(.+?)(?:\s*·\s*(.+))?$/
      Captures: [1]=source, [2]=meta, [3]=timestamp
    
    Tags line:
      /#([^`]+)/g
      Captures: tags between # and backtick

  Lines Skipped:
    - <a id=...> (anchor tags)
    - <details> (collapsible sections)
    - <ul>, <li>, </...> (HTML lists)
    - --- (separators)
    - > ... (blockquotes)
    - /^\d+\.\s*\[/ (TOC entries)

  Extraction:
    - Title from [...]
    - URL from (...)
    - Score from score/10
    - Sources from "source · meta" lines
    - Background from **Background**: block
    - Discussion from **Discussion**: block
    - Tags from **Tags**: line

✅ STRENGTHS:
  • Zero backend complexity (pure static HTML)

  • Manual run of conversion script (no automation)

🎯 GOOD FOR:
  • Curated daily digests
  • Chronological browsing
  • Social sharing
  • SEO (static HTML)

❌ NOT SUITABLE FOR:
  • Real-time updates
  • Complex filtering
  • Full-text search
  • User interactions
