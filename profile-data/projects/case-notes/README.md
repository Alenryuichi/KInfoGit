# 项目文档目录（case-notes）

这个目录用于存放每个「核心项目」的详细文字说明，方便后续生成 `/work/[id]` 的完整 case study 页面。

## 命名约定

- 每个文件对应一个项目，**文件名 = 项目 id + `.md`**。
- 项目 id 来自 `profile-data/projects/core-projects.json` 里的 `id` 字段，例如：
  - `portrait-platform` → `portrait-platform.md`
  - `anti-fraud-governance` → `anti-fraud-governance.md`

## 内容建议

- 语言：推荐先用中文写，口语一点没关系，我会帮你整理成适合 HR / 面试官阅读的版本。
- 结构：可以参考同目录下的 `_template.md`。
- 风格：更像讲故事 / 复盘，而不是仅仅列 bullet。

## 后续我会做的事

- 你把若干项目文档写好放进来之后，我会：
  1. 扫描这些 Markdown；
  2. 抽取「背景 / 角色 / 方案 / 结果 / 反思」等关键信息；
  3. 更新 `/work/[id]` 的页面结构，让有文档的项目显示更完整的 case study。
