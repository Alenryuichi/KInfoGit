## ADDED Requirements

### Requirement: Sync engine authenticates via Yuque API token
The sync script SHALL authenticate to Yuque API using an `X-Auth-Token` header with the value from `YUQUE_TOKEN` environment variable. The script SHALL NOT use username/password authentication.

#### Scenario: Valid token
- **WHEN** `YUQUE_TOKEN` is set and valid
- **THEN** the script successfully authenticates and proceeds with sync

#### Scenario: Missing token
- **WHEN** `YUQUE_TOKEN` is not set
- **THEN** the script exits with error code 1 and prints an error message

### Requirement: TOC parsing extracts one-level directory structure
The sync script SHALL call `GET /api/v2/repos/:namespace/toc` and parse the response to identify first-level directories (nodes with `type=TITLE` and `depth=1`). Each first-level directory title SHALL become the `category` value for all documents nested under it.

#### Scenario: Three first-level directories
- **WHEN** the TOC contains directories 文章, 分享, 随笔 at depth=1
- **THEN** the script identifies three categories: 文章, 分享, 随笔

#### Scenario: Document nested under a directory
- **WHEN** a document with `type=DOC` appears at depth=2 under the 文章 directory
- **THEN** the document's category is set to "文章"

#### Scenario: Document not under any directory
- **WHEN** a document appears at depth=1 (not nested under a TITLE)
- **THEN** the document's category is set to "未分类"

### Requirement: Category order preserved from TOC
The sync script SHALL record the order of first-level directories as they appear in the TOC response. Each blog post frontmatter SHALL include a `categoryOrder` field (integer, starting from 0) reflecting this order.

#### Scenario: TOC order is 文章, 分享, 随笔
- **WHEN** the TOC lists directories in order: 文章, 分享, 随笔
- **THEN** posts under 文章 have `categoryOrder: 0`, 分享 have `categoryOrder: 1`, 随笔 have `categoryOrder: 2`

### Requirement: Document content fetched via API
The sync script SHALL fetch each document's content by calling `GET /api/v2/repos/:namespace/docs/:slug` and extracting the `body` field (markdown content).

#### Scenario: Successful fetch
- **WHEN** the API returns a document with `body` containing markdown
- **THEN** the script uses the body as the document content

#### Scenario: API rate limiting
- **WHEN** multiple documents need fetching
- **THEN** the script inserts a delay (at least 500ms) between API requests

### Requirement: Document date uses Yuque created_at
The sync script SHALL use the document's `created_at` field from Yuque API as the blog post `date` in frontmatter, formatted as `YYYY-MM-DD`.

#### Scenario: Document created on 2026-03-15
- **WHEN** a document has `created_at: "2026-03-15T10:30:00.000Z"`
- **THEN** the blog post frontmatter `date` is "2026-03-15"

### Requirement: Images downloaded to local directory
The sync script SHALL scan document content for image URLs, download each image to `website/public/blog/images/`, and replace the URL in markdown with the local path `/blog/images/<filename>`.

#### Scenario: Yuque CDN image in content
- **WHEN** document content contains `![alt](https://cdn.nlark.com/yuque/xxx/image.png)`
- **THEN** the image is downloaded to `website/public/blog/images/<doc-slug>-<hash>.png`
- **AND** the markdown is updated to `![alt](/blog/images/<doc-slug>-<hash>.png)`

#### Scenario: Image download fails
- **WHEN** an image URL returns an error
- **THEN** the script logs a warning and keeps the original URL

### Requirement: HTML tags cleaned from content
The sync script SHALL remove HTML tags from document content (font, span, div, p, br, anchor, style attributes, comments) while preserving the text content within them.

#### Scenario: Content with font tags
- **WHEN** document contains `<font color="red">important</font>`
- **THEN** the output contains `important` without the font tags

### Requirement: DeepSeek generates tags and excerpt
The sync script SHALL call DeepSeek API to generate `tags` (3-5 tags) and `excerpt` (50-100 characters) for each new document. The `category` field SHALL NOT be generated by DeepSeek — it comes from the TOC directory.

#### Scenario: New document synced
- **WHEN** a document is synced for the first time
- **THEN** DeepSeek is called to generate tags and excerpt
- **AND** the category is set from the TOC directory name, not from DeepSeek

#### Scenario: Previously synced document updated
- **WHEN** a document's `updated_at` has changed but it was previously synced
- **THEN** tags and excerpt are re-generated by DeepSeek

### Requirement: Incremental sync via state cache
The sync script SHALL maintain a `sync-state.json` file mapping `doc_id` to `{ slug, updated_at, category, blogFile }`. On each run, it SHALL compare the remote `updated_at` with the cached value and only process documents that are new or updated.

#### Scenario: Document unchanged
- **WHEN** a document's `updated_at` matches the cached value
- **THEN** the document is skipped

#### Scenario: New document
- **WHEN** a document's `doc_id` is not in the cache
- **THEN** the document is fully processed and added to the cache

#### Scenario: Cache file missing
- **WHEN** `sync-state.json` does not exist
- **THEN** all documents are treated as new (full sync)

### Requirement: Blog post frontmatter schema
Each generated blog post SHALL have frontmatter with: `title` (string), `date` (YYYY-MM-DD from created_at), `tags` (string array from DeepSeek), `category` (string from TOC directory), `categoryOrder` (integer), `readTime` (string), `featured` (boolean, default false), `excerpt` (string from DeepSeek).

#### Scenario: Generated frontmatter
- **WHEN** a document titled "Page Agent 分析" under directory "文章" (order 0) is synced
- **THEN** the frontmatter includes `category: "文章"`, `categoryOrder: 0`, and `date` from Yuque `created_at`
