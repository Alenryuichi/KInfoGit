# Yuque Sync - Quick Reference Guide

## At a Glance

| **Aspect** | **Blog Sync** | **Work Sync** |
|-----------|--------------|--------------|
| **Script** | `scripts/sync-yuque.ts` | `scripts/sync-yuque-work.ts` |
| **Source** | Yuque Blog Repo (qd9got) | Yuque Work Repo (sh4e9k) |
| **Output** | `profile-data/blog/*.md` | `profile-data/projects/*.md` + `core-projects.json` |
| **Schedule** | Daily 00:00 UTC | Monday 01:00 UTC |
| **State File** | `tools/yuque-sync/sync-state.json` | `tools/yuque-sync/work-sync-state.json` |
| **Images** | `website/public/blog/images/` | `website/public/work/images/` |
| **AI Processing** | Tags + excerpt | Structured extraction (title, role, tech_stack, etc.) |
| **Delay Between Docs** | 500ms | 1500ms |
| **Featured** | Manual (featured: true/false in frontmatter) | Auto-assigned (top 3 by order) |

### Manual Sync (Local Dev)
```bash
# Ensure env vars set
export YUQUE_TOKEN=xxx
export DEEPSEEK_API_KEY=xxx

# Blog sync
npx tsx scripts/sync-yuque.ts

# Work sync
npx tsx scripts/sync-yuque-work.ts

npx tsx scripts/sync-yuque-work.ts --force
```

### Force Sync via GitHub Actions
1. Go to `.github/workflows/sync-yuque.yml` → Run workflow
2. Check "force_sync" checkbox
3. Or manually trigger from GitHub UI with workflow inputs

## Yuque API Reference

### Endpoints Used

```
GET /repos/{namespace}/toc
    └─ Returns: TOC structure (categories, documents)

GET /repos/{namespace}/docs?offset=0&limit=100
    └─ Returns: Document list (for checking updated_at)

GET /repos/{namespace}/docs/{slug}
    └─ Returns: Full document (body HTML + metadata)
```

### Rate Limits
- **Hour quota**: Check `x-cchm` header (e.g., `h:5,m:30`)
- **Per-second quota**: Check `X-RateLimit-Remaining` header
- **429 response**: Rate limited; retry with backoff

### Required Header
```javascript
'X-Auth-Token': YUQUE_TOKEN  // Yuque API authentication
```

## Image Handling

### Naming Convention
```
{doc-slug}-{8-char-md5-hash}.{extension}

Examples:
- article-title-a1b2c3d4.png
- betaline-project-e5f6g7h8.jpg
```

### Process
1. Extract markdown images: `![alt](https://...)`
2. Download from remote URL
3. Hash content with MD5
4. Save locally
5. Rewrite reference: `![alt](/blog/images/filename)` or `![alt](/work/images/filename)`

### Error Handling
- Failed downloads logged as warnings
- Sync continues without aborting
- Original references preserved if download fails

## Troubleshooting

### "YUQUE_TOKEN 环境变量未设置"
**Problem**: Missing API token
**Solution**: 
- Set in GitHub Actions secrets
- Or create `tools/yuque-sync/.elog.env`
- Get token from: https://www.yuque.com/settings/tokens

### "Yuque API 429: 配额耗尽"
**Problem**: Rate limit hit (hour quota exhausted)
**Solution**:
- Wait for next hour
- Or reduce between-doc delays
- Check `x-cchm` header in error message

### "DeepSeek 调用失败"
**Problem**: AI extraction failed
**Solution**:
- Check `DEEPSEEK_API_KEY` is set
- Verify API key has sufficient credits
- Falls back gracefully to defaults (tags=['未分类'])

### "No images found" or images not downloading
**Problem**: Image URLs changed or invalid
**Solution**:
- Check Yuque document still has images
- Verify URL formats are correct
- Check image permissions in Yuque
- Logs will show which images failed

### "Repository is out of sync"
**Problem**: Local files not matching Yuque
**Solution**:
```bash
# Clear sync state (full resync next run)
rm tools/yuque-sync/sync-state.json
# Or via GitHub Actions: run with force_sync=true

# Then trigger sync
npx tsx scripts/sync-yuque.ts
npx tsx scripts/sync-yuque-work.ts --force
```

### "undefined" in project titles or fields
**Problem**: DeepSeek extraction failed
**Solution**:
- Force sync to retry extraction
- Check document content is in Chinese/English
- Verify DeepSeek prompt compatibility
- Check error logs for details

## Common Workflows

### Update Blog Article
1. Edit in Yuque
2. Wait for scheduled sync (daily 00:00 UTC)
   - OR manually trigger via GitHub Actions
3. Article updated on website (~10 min after sync)

### Force Resync All Content
1. Go to `.github/workflows/sync-yuque.yml` or `.sync-yuque-work.yml`
2. Click "Run workflow"
3. Check "force_sync" (or "force_sync" + pass `--force`)
4. Sync states cleared, all docs reprocessed
5. Website rebuilt and deployed

### Add New Category/Subcategory
1. Create new section/subsection in Yuque TOC
2. Add document under new category
3. Run sync (manual or wait for schedule)
4. Blog filters auto-update with new category

### Change Project Order/Featured Status
**Projects**:
- Order controlled by position in Yuque TOC
- Featured status auto-assigned: top 3 → featured=true

**Blog**:
- Manually edit frontmatter: `featured: true/false`
- Or set post date to control sort order

### Debug Failed Sync
1. Check GitHub Actions log
2. Look for specific error messages
3. Common issues:
   - Rate limit (wait 60 min)
   - API token invalid (check secrets)
   - DeepSeek failure (check API key, retry)
   - Image download failure (non-blocking, warning only)

## Performance Notes

### Sync Duration
- **Blog**: 2-5 minutes (depends on # docs + AI processing)
- **Work**: 3-8 minutes (heavier DeepSeek processing)

### Optimization Tips
- Blog sync: 500ms delay between docs (tunable)
- Work sync: 1500ms delay (stricter, don't reduce)
- Batch doc list query: 1 API call for all doc metadata (fast)
- Incremental sync: Only changed docs reprocessed (fast)
- Parallel image downloads: Not implemented (sequential, potential bottleneck)

### Cost
- **Yuque API**: Free tier (limited quota per hour)
- **DeepSeek API**: Billed per token (~0.5-2 cents per blog article)
- **GitHub Actions**: Free tier (up to 2,000 min/month)
- **GitHub Pages**: Free (static hosting)
